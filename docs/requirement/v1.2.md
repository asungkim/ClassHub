# 프로젝트 구상

# 1. 문제 정의 및 목표 설정

## 📘 프로젝트 개요

- **프로젝트명:** ClassHub (임시명)
- **목표:** 학원 강사들이 수업, 학생, 조교 일정을 통합 관리하고 커뮤니케이션을 효율화하는 플랫폼을 구축한다.

## 💡 배경 및 필요성

- **기존 방식(노션, 엑셀)의 문제점**
  - 공지사항 전달을 위해 카카오톡 등으로 반복적으로 공유해야 함
  - 수업 현황 및 학생별 진도 관리의 체계 부족
  - 클리닉 출석 체크 누락, 기록 관리 어려움 → 결석자 관리 힘듬
  - 노션 페이지가 많아질 경우 비용 부담 발생
- **시장조사**
  - 현재 학원 강사용 일정·조교 관리 전용 통합 서비스는 부재함
  - 기존 노션, 구글폼, 엑셀 등을 임시 대체로 사용하는 수준

## (3) 사용자 흐름 (User Flow)

- Admin 은 시스템으로 생성
- Teacher 는 회원가입을 통해 로그인
- Assistant 는 Teacher의 초대를 통해 회원가입/로그인
- Student 는 Assistant의 초대를 통해 회원가입/로그인

### 👩‍🏫 선생님(Teacher)

1. 메인화면에 업무 내용을 대시보드 형태로 제공된다.
2. 메인화면에서 대시보드를 커스텀화 할 수 있다. (정해진 소계만)
3. 반을 생성할 수 있다.
4. 반별로 작성한 공통진도를 조회, 수정, 삭제할 수 있다.
5. 학생들의 목록을 볼 수 있다.
6. 학생별 페이지를 볼 수 있다.
   1. 캘린더 형태로 제공한다.
   2. 반에서 진행한 공통진도, 개별로 나간 개별진도, 클리닉 내용 3가지를 다른 색깔로 표시하여 캘린더에 표기
7. 수업 내용을 작성한다.
   1. 수업 내용 작성 버튼을 누르면, 반을 선택하여 공통진도 내용을 작성하고 학생별로 선택하여 개별 진도 내용을 작성한다.
   2. 공통진도 내용은 수업 내용 작성시마다 필수 입력, 개별진도는 선택 사항
8. 조교를 초대한다.
9. 조교 전용 공지사항을 작성한다.
   1. 조교 별로 읽었다는 걸 체크할 수 있게 한다.
10. 모든 학생의 클리닉 내용을 작성, 조회, 수정, 삭제 할 수 있다.
11. 조교들의 근무일지를 조회한다.
12. 클리닉 시간표를 작성, 조회, 수정, 삭제 할 수 있다.
13. 오늘 클리닉의 세션 정보 확인하고 학생들의 ClinicRecord를 작성, 조회, 수정, 삭제할 수 있다.

### 🧑‍💼 조교(Assistant)

1. 오늘 클리닉의 세션 정보 확인하고 학생들의 ClinicRecord를 작성, 조회, 수정, 삭제할 수 있다.
2. 학생들을 초대한다.
3. 근무일지를 작성, 조회, 수정, 삭제 할 수 있다.

### 🧑‍💼 학생(Student)

1. 할당된 클리닉 시간을 변경 할 수 있다.

### 🧑‍💻 관리자(Admin)

- 전체 권한 보유

## (4) 기능 요구 정의 방식

- 상세 기능 요구사항(FR)은 Phase별 TODO를 진행하면서 각 작업의 PLAN 문서에서 정의한다.
- Requirement 문서는 사용자/Use Case/흐름/비기능 기준만 유지하며, 구현 세부사항은 `docs/plan/<feature>_plan.md`에 기록 후 스펙/ TODO로 반영한다.
- 기능 추가나 변경이 필요하면 **반드시** 이 문서 → 스펙 → TODO 순으로 버전을 올린다.

## (5) 비기능 요구사항

| 분류                     | 요구사항                                            |
| ------------------------ | --------------------------------------------------- |
| **보안(Security)**       | JWT 기반 인증 / Role 기반 접근제어 / HTTPS 통신     |
| **성능(Performance)**    | 평균 응답속도 1초 이하, 50명 동시접속 기준          |
| **확장성(Scalability)**  | 사용자 및 데이터 증가 시 수평 확장 가능 구조        |
| **가용성(Availability)** | Docker 기반 컨테이너 배포, 장애 시 5분 내 복구 목표 |
| **데이터(Data)**         | MySQL 사용, Redis 캐싱으로 조회 속도 향상           |
| **UI/UX**                | 반응형 웹 지원 (PC/Tablet/Mobile)                   |
| **운영(Ops)**            | CI/CD 구축 (GitHub Actions → Docker → AWS EC2 배포) |

## (3) 도메인 설계

### 주요 엔티티

| 엔티티         | 설명                                                   |
| -------------- | ------------------------------------------------------ |
| **Member**     | 사용자 엔티티(Teacher, Student, SuperAdmin, Assistant) |
| Course         | 반 정보(Teacher가 생성)                                |
| SharedLesson   | 공통 진도                                              |
| CliicSlot      | 클리닉 시간표(요일, 시간)                              |
| Notice         | 선생님이 조교에게 전달하는 공지사항                    |
| Invitation     | 선생님이 조교를, 조교가 학생을 초대                    |
| ClinicSession  | 매주 열리는 클리닉 세션                                |
| ClinicRecord   | 선생님, 조교가 기록하는 클리닉 내용                    |
| PersonalLesson | 학생별 개별 진도                                       |
| StudentProfile | 학생 상세 정보                                         |

### 관계

| **관계**                                   | **설명**                         |
| ------------------------------------------ | -------------------------------- |
| Member 1:N Course (Teacher)                | 선생님은 여러 반을 관리          |
| Member 1:N Notice (Teacher)                | 공지사항 작성 (조교 대상)        |
| Member 1:N Invitation (Teacher, Assistant) | 초대 생성                        |
| Member 1:N ClinicSlot (Teacher)            | 클리닉 시간표 생성               |
| Member 1:N StudentProfile                  | 선생님이 학생 상세 정보 생성     |
| StudentProfile 1:N PersonalLesson          | 학생 정보에 개별 진도내용 포함   |
| ClinicSlot 1:N ClinicSession               | 특정 날짜 실제 클리닉 세션       |
| ClinicSession 1:N ClinicSessionStudent     | 특정 세션 학생 목록              |
| Member 1:N ClinicSessionStudent (Student)  | 특정 클리닉 세션에 참석하는 학생 |
| ClinicSessionStudent 1:1 ClinicRecord      | 한명의 학생에게 한명의 기록      |
